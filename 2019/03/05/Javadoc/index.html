Javadoc

Why do we need Javadoc?

Link the code to the documentation, put everything in the same file.
In order to complete this picture, you need a special comment syntax to mark the documentation and a tool to extract those comments and put them in a useful form.
This is what Java does.

What is Javadoc

Javadoc is the tool to extract the comment, and it’s part of JDK installation. It uses some technology from Java compiler to look for special comment tags you put in your programs
It not only extract the information marked by the tags, it also pulls out the class name or method name that adjoins the comment.

Output of Javadoc

The output of Javadoc is HTML file that you can view with your Web browser.

Doclets (your own Javadoc handlers)

If you want to perform special operations on the information processed by Javadoc (to produce different format, example)
Doclets are introduced in the supplement at BetterJava

Syntax

There are three types of comment documentation:

  1. class
  2. field
  3. method

NOTE that Javadoc will processed documentation for only public and protected members. Comments for private and package-access members are ignored, you will see no output.
* however, you can use the (-private) flag to include private members.
