Javad环境变量浅析

Javad环境变量有三种

  1. PATH
  2. CLASSPATH
  3. JAVA HOME

首先搞清楚一个基础概念:

current directory （当前目录）：见名知意，当前在用的目录就是当前目录

比如说，你打开Notepad，并处于运行状态的时候，当前目录就是c:/windows;
所以当你在使用java命令时，当前目录就是JAVA里面BIN目录，因为java.exe这个指令就在bin目录里面
*需要注意的是，如果我们不进行classpath的环境变量配置，系统默认路径就是当前目录 这也是为什么如果你不设置环境变量，只要你cd到BIN目录也可以使用javac,java.*

搞清楚这个基础概念之后我们来看第一个环境变量配置

path
设置path是为了在系统中的任何地方都可以运行JDK中的java工具（例如javac,java,javap etc）
系统变量path告诉OS可执行文件(.exe, **.bat etc)所在的路径

当OS发现某个.exe的时候，默认从当前目录开始找这个命令，如果在当前目录里找不到，就去path设定的路径中去找。

系统默认的系统变量是：Path = %SystemRoot%;%SystemRoot%/system32;%SystemRoot%/System32/Wbem
这个意思是说，处于上面三个目录（用分号隔开的）里的.exe文件，可以在任何地方被执行。
这样的话，你在运行窗口里就可以直接执行cmd,notepad etc.
那么问题来了，这个%SystemRoot%是啥意思呢？

%SystemRoot%就是在安装OS的时候，系统默认的安装路径

举个栗子吧
如果你的windows装在C:/WINDOWS
那么你的%SystemRoot%就是c:/windows
%SystemRoot% 只是一个符号，代表你的系统安装目录
当我们在进行java开发的时候，OS需要用java.exe,javac.exe等，所以为了方便，我们要把JDK所在路径里的bin加到path里去

搞清楚path之后我们再来看第二个环境变量

classpath:
什么是classpath?
CLASSPATH 是专门针对java的，例如windows的path；path是针对整个windows
鉴于现在还没有学到JVM，只是简单带过了，CLASSPATH就是告诉java虚拟机(JVM)要执行的.class文件在哪
所谓JVM，就好像在MS OS上面再激活多一个OS，对于JVM来说，CLASSPATH就如同PATH对于MS OS
所以呢，你要用JVM开运行程序，就必须有个CLASSPATH告诉它去哪开
JVM和windows一样，都有个默认的查找路径，就是当前路径，对于初学者来说，不设置也可以。

那么现在我们看最后一个环境变量

JAVA_HOME：
什么是JAVA_HOME？
JAVA_HOME是指JDK的安装目录，在这个路径下你可以找到bin,lib等等目录
那么问题来了，JDK安装目录要来干嘛，为什么要用它？不用行不行？
答案是不用也是OK的，但是最好用一下，日后当我们要改变某个JDK的时候只需要改JAVA_HOME的值就好了。
等后面看了Tomcat的启动分析你就明白了
**在环境变量里引用它的时候用%JAVA_HOME%来表示
