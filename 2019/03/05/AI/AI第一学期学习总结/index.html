AI第一学期学习总结

Agents Acting in Environment


What is an agent?

  • in some environment
  • can observe (have sensors)
  • can act (have effectors)
  • has goal and preferences
  • may have prior knowledge or beliefs, and some way of updating it’s beliefs based on new experience

Rational Agent

What is rational agent?
Basically rational agent is one that does right thing

Rational action: action that maximizes the expected value of an objective performance

Rationality:

  1. Performance measure
  2. Prior knowledge about the Environment
  3. Everything that agent learn through sensors
  4. Actions that can be performed

Some example of consideration about rationality:
Driverless Car

Performance measure:

  1. safety
  2. reach destination
  3. obey laws

Environment:

  1. Singapore urban streets
  2. highways
  3. traffic

Sensors

  1. speed
  2. engine status
  3. GPS

Actions

  1. accelerate
  2. brake
  3. horn

Medical Diagnosi System

Performance measure:

  1. healthy patient
  2. faster recovery
  3. minimize costs

Environment:

  1. patient
  2. hospital
  3. clinic

Sensors

  1. symptoms
  2. findings
  3. patient’s answers

Actions

  1. questions
  2. medical tests
  3. treatments

Types of Environment

  • Accessible/inaccessible

    sensor can sense the complete state of environment

  • Multi agent/single agent

    adversary or cooperative

  • Deterministic/stochastic

    next state of the environment completely determined by the current state + actions selected by the agent
    it’s stochastic if
    Sensing Uncertaninty:cannot sense whole current state of world
    Effect Uncertaninty: agent know the effects of the actions?

  • Static/dynamic

    environment does not change while agent is deciding

  • Discrete/continuous

    limited number of distinct percepts and actions

  • Episodic/sequential

    each episode is not affected by previous actions

How to determine the above environment attributes?
Some examples

  1. poker is stochastic because players play based on assumptions
  2. Tossing a coin is not deterministic
  3. Chess is deterministic because when it is your move, you know fully the state of the game(environment) and what action will take the game to what state

Here is some example for you to better understanding these concept:
Driverless Taxi

  1. Accessible?
    No, some traffic info on the road is missing (cannot sense the whole picture for all reasons)
  2. Deterministic?
    No, cars in front may turn right suddenly (Next state of environment is not only depend on current state and my action,it also depend on other cars)
  3. Episodic?
    No, current action is based on previous driving actions
  4. Static?
    No, when taxi moves, other cars also moves.
  5. Discrete?
    No, the speed, distance and fuel consumption are in real domains, not limited number of distinct percepts.

Chess

  1. Accessible?
    Yes, all postion can be observe.
  2. Deterministic?
    Yes, after each move, the whole configuration can be determined.
  3. Episodic?
    No, the current action will depend on previous movements.
  4. Static?
    Yes, when you are considering next step, other people can’t move.
  5. Discrete?
    Yes, all positions and movements are in discrete domains.

agent program

What is agent program?
A function that implements the mapping from percepts to actions.

  1. sensor percept and update memory
  2. base on the memory choose the best action
  3. understand what my action will change the world and update it to my memory
    Then return the action I will be perform

Different agent program

Simple Reflex Agent

  1. Sense the current world
  2. Find the rule matches the current situation
  3. Perform the action associated with that rule

    example: if car in front is braking then initiate braking.

Reflex agent with state

  1. Sense the current world and update agent’s state
  2. Based on the state to find the best rule to match
  3. perform action according to the rule
  4. update the state with the action effect to the world

    example: if yesterday at NTU and no traffic jam now then go orchad

Goal-based agent

example: At a known state, should I go left,right or straight? To reach Orchard

Utility-based agent
There are many way can achieve the same goal, which way should I go for?

exmaple: go orchard(destination) via PIE? or AYE? Which one charges lower fare?

You said agent program is representation & reasoning, what is that?

What do we need to represent?

  1. The environment: what different configurations can the world be and how to denote them?
  2. How the world works?
  • Actions: precondition and effects
  • Constraint: sum of current into a node = 0(exmaple)
  • Causal: what are the causes and effects of brain disorders?

Corresponding reasoning tasks

What are the tasks?

  1. planning: find sequence of actions to reach the goal state/maximize utility
  2. Constraint satisfaction: Find state that satisfies set of constraints
  3. Answering query: Is a given proposition true?

(Goal based) problem solving agent

Rational goal-based agent:
Performance measure defined in terms of satisfying goals

Steps:

  1. goal formulation (define goal states)
  2. problem formulation (define what states and actions to consider)
  3. Search for a solution (sequence of actions that lead to goal state)
  4. Execution

Problem formulation
Five components:

  1. Initial state
  2. Actions
  3. Successor function (result of each action)
  4. Goal test
  5. Cost

How can we find a solution?

  1. Define search space graph (nodes are states, edges are actions)
  2. Explore search graph by generating successors o explored states.

    Frontier are candidate nodes for expansion

Evaluation of search strategies

  1. Completeness (does it always find a solution if there is one?)
  2. Time complexity
  3. Space complexity
  4. Optimality

Branching factor: Maximum number of successors of any node

Unninformed Search (lecture 2b)

Uninformed search strategies

  • Breadth-first search (BFS)
  • Uniform-cost search (UCS)
  • Depth-first search (DFS)
  • Depth-limited search (DLS)
  • Iterative deepening search (IDS)

!!Different strategies are differnt with respect to how they add/remove paths from the frontier

Informed search
Use problem-specific knowledgeto guide the search, usually more efficient

BFS

Queue, FIFO from frontier
max path length is m, max branching factor is b
Time complexity = O(b^m)
Space complexity = O(b^m)
BFS is complete
BFS is optimal if the steps-costs are equal

Uniform-Cost search

Priority queue from frontier
UCS is a special form of BFS, it also called lowest cost search
UCS expand paths from the frontier in order of their costs
Edge cost must be greater than zero in order to use the UCS

DFS

Stack (LIFO)
DFS is not complete (will stuck if there is a cycle)
DFS is not optimal (it will stumble on longer solution paths before it gets the shorter one)
Time complexity = O(b^m)
Space complexity = O(bm)
Why DFS has linear space complexity?

  1. start with root, push 1,2,3….b to the stack
  2. pop 1, push 11,12,13….1b to the stack
  3. pop 11, push 111,112,113…..11b to the stack
    …..
    …..
    So every height, there will be (b-1) nodes in the stack
    Total height is m, hence the space complexity is O(bm)

DLS Depth-limited search

To avoid infinite loops, DLS is DFS with a cutoff on the max depth l of a path
Is complete only when l is greater than depth of shallowest goal
Time: O(b^l)
Space: O(bl)
DLS is not optimal

IDS Iterative Deepening Search

Goal of IDS: achieve linear spcae complexity, complete and optimal
Key idea: re-compute elements of the frontier rather than saving them
How to understand this key idea?
Iterative estimate the max depth l of DLS one by one

Time complexity of IDS: O(b^m)
How to get the time complexity?
depth1 total number of path is b, and in IDS it will be created m times
depth2 total number of path is b^2, and it will be created m-1 times
….
depthm total number of path is b^m, and it will be created one time only
So in total there will be O(b^m)
